## Notes On Semantics

- Several wrappers are intentionally no-op in JS because returned data is already copied into JS-owned memory and native pointers are not exposed directly.
- Memory helper wrappers provide JS-friendly behavior (`ArrayBuffer`), not raw pointer lifecycle semantics.
